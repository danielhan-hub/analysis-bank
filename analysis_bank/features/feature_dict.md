# Analysis Feature Dictionary (76 features)

This is the canonical scoring rubric used by `analysis_bank.features.scorer`. Every accepted procedure and every analyst question is scored on these 76 integer features, range **−5..+5**.

The rubric is **unified across questions and analyses**. When scoring a question, score it as if you were scoring the SQL that would best answer it — what would that SQL emphasize, scope, control for, or compute? When scoring an analysis (procedure.sql + README), score what is actually present.

`analysis_id` is metadata only and is not a feature.

| canada_market_focus                              | Degree to which the analysis is scoped to Canada (warehouse_country_id = 124, CA accounts/retailers).                    | Explicit US-only scope with no CA logic.                                                            | CA-only scope with country_id = 124 or CA-specific account_id at the core of the analysis.                                | No country scope, or country appears only as a passthrough column.                     |

| theme_activation_vs_optimization                 | Whether the analysis is shaped around activating/launching new tactics vs optimizing existing programs.                  | Pure optimization of an established program (efficiency, pacing, refinement).                       | Pure activation: feasibility, launch readiness, or new tactic adoption is the core question.                              | Mixed or unclear (e.g., both pre-existing and new tactics analyzed together).          |

| theme_portfolio_allocation                       | Multi-campaign / portfolio-level allocation tradeoffs.                                                                   | Single campaign or single line item only.                                                           | Portfolio-wide rebalancing across many campaigns/brands/tactics with explicit tradeoffs.                                  | Multi-campaign data present but no tradeoff math.                                      |

| theme_long_term_effects_retention                | Long-run, retention, halo, or repeat effects (vs short-term immediate outcomes).                                         | Only same-window/short-term outcomes.                                                               | Explicit long-horizon look-forward windows (e.g., 26+ week, retention curves, LTV proxy).                                 | Some look-forward but window is moderate (≤4 weeks) or peripheral.                     |
| theme_measurement_methodology_validation         | Validating, debugging, or stress-testing measurement itself (rather than measuring an outcome).                          | Assumes the measurement stack is correct; pure outcome.                                             | Core purpose is to validate methodology — A/A test, monte-carlo fairness, DiD validation, sensitivity check.              | Some sanity checks but methodology is not the focus.                                   |
| theme_framework_or_playbook                      | Designed as a reusable framework/template vs a one-shot bespoke pull.                                                    | Pure ad-hoc one-shot with hardcoded values.                                                         | Heavily templated/parameterized for reuse (e.g., $account_id, $period_end_date, stored procedure form).                   | Some parameters present but mostly hardcoded.                                          |

| rca_orientation                                  | Root-cause investigation orientation (anomaly, drop, miss, or unexpected behavior chase).                                | No RCA framing; not investigating a problem.                                                        | Core mission is RCA — investigates a measured drop/miss with hypothesis-by-hypothesis logic.                              | Some diagnostic columns but not framed as RCA.                                         |

| focus_sp                                         | Focus on Sponsored Products.                                                                                             | SP not involved at all.                                                                             | SP outcomes (SP impressions, agg_featured_product_daily, SP MTA) are the core surface analyzed.                           | SP appears but is one of several products.                                             |
| focus_suas                                       | Focus on SUAS / Single Unit Auto Savings.                                                                                | No SUAS concept; unrelated.                                                                         | SUAS tests, redemption, or ad behavior is the core subject.                                                               | SUAS mentioned but not core.                                                           |

| focus_coupons_free_gift                          | Focus on coupons, free gifts, or promotion mechanics.                                                                    | No coupons/free gifts; pure media analysis.                                                         | Coupon/free gift mechanics (nexus_coupons, fact_spend_promotion_redemption, discount_policies) are core.                  | Promo data present but not central.                                                    |

| design_experimental_vs_observational             | Position on observational ↔ experimental spectrum.                                                                       | Fully observational; no randomization or designed exposure.                                         | Cleanly designed randomized experiment at the core (assignment table + treatment/control contrast).                       | Mixes designed-experiment data with observational analysis.                            |
